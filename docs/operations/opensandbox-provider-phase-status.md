Operator configuration:
- Self-hosted/local OpenSandbox API: set `SANDBOX_CONTAINER_PROVIDER=opensandbox`, `OPENSANDBOX_DOMAIN=<host>:<port>`, `OPENSANDBOX_PROTOCOL=http`, and leave `OPENSANDBOX_API_KEY` empty unless that server requires auth.
- Managed/cloud API: set `OPENSANDBOX_DOMAIN=api.opensandbox.io`, `OPENSANDBOX_PROTOCOL=https`, and provide `OPENSANDBOX_API_KEY` only in the target runtime environment.
- The adapter supports OpenSandbox private endpoint headers internally for executor health checks and dispatch. These headers must not be serialized to DB lease payloads, Admin Runtime projections, events, or evidence.
- Real sandbox executors receive only server-side configured model/API credentials as container environment variables so the Claude Agent SDK can run inside the isolated executor; user payloads cannot choose this provider or inject these values. The mounted workspace root is forced to the sandbox container path rather than the host workspace path.
- OpenSandbox may return endpoint strings without a scheme, such as `<host>:<port>/proxy/18000`; the adapter normalizes those to `OPENSANDBOX_PROTOCOL://...` before health checks and executor dispatch.
- The executor image defaults to `SANDBOX_EXECUTOR_IMAGE`; `OPENSANDBOX_EXECUTOR_IMAGE` can override it when the OpenSandbox data plane needs a different image tag.
- The 2026-07-08 label-clean SDK smoke sourced model/Claude SDK settings from a historical 211 deploy environment path without printing values. The repo-local deploy environment path had no SDK/model variables, so this is valid first-stage smoke evidence but not G0 source-authority closure.
- For controlled B2 verifier runs that need Claude Agent SDK evidence inside the sandbox executor, the verifier environment must explicitly enable `CLAUDE_AGENT_SDK_ENABLED=true`; the default API service environment intentionally does not turn on the worker-only SDK switch.
- Platform smoke generation must send a configured runtime model to `SandboxRuntimeRequest.model`; `smoke` is only a verifier label concept and is not valid Claude Agent SDK model input.
- On the current 211 OpenSandbox data plane, controlled callback receivers must use a callback public URL reachable from the sandbox, such as the 211 host IP, and that host must be included in `OPENSANDBOX_ALLOWED_EGRESS_HOSTS`. `host.docker.internal` remains the API/worker default gateway label but was not sufficient for the d4b43ba OpenSandbox callback path.
- On the current 211 temporary Docker-runtime OpenSandbox server, the SDK policy object is not sufficient hardening evidence. B2 hardening needs a data plane that actually denies unapproved outbound traffic, provides resource-limit enforcement evidence equivalent to Docker inspection plus over-limit cleanup, and proves non-privileged/minimal capability/root filesystem posture. The Docker-runtime `secure_access` flag is not an available workaround on this host.
