# 来源边界

登记来源文件、生成来源索引或检查证据时使用本参考文件。

## 允许来源

允许来源是用户在当前抽取任务中提供或明确批准可用于事实的文件。每个抽取事实都必须追溯到一个或多个允许来源，或追溯到由允许来源派生且已审查的中间产物。

每个来源应记录：

- 稳定的 `source_id`；
- 路径；
- 文件类型；
- 使用的 reader 或 parser；
- 解析/OCR 状态；
- 是否允许用于事实；
- 警告或审查说明。

## 禁止来源

禁止来源必须登记，但不得作为事实证据使用。默认禁止类别包括：

- `~$...` 等临时锁文件；
- 既往最终输出或已完成申报资料，除非用户明确批准其作为事实来源；
- 示例、模板、脚本、写作指南和 fixtures，除非用户明确提升为来源材料。

Profile 可以添加更严格的禁止来源模式。CTD 稳定性 profile 默认禁止将已完成申报资料作为项目事实来源。

## Source-index 规则

`source-index` 是从来源文件到定位信息的地图。它帮助抽取角色找到候选证据，但不能替代来源证据。

事实应引用原始来源，或引用已审查的中间产物，并提供足够定位信息以便复核：

- PDF 页码 / 表格 / 章节；
- DOCX 段落或表格索引；
- XLSX sheet / 行 / 列 / 单元格范围；
- OCR manifest、页码、表格和已审查文本；
- 文本行号或章节。

## OCR 规则

如果 PDF 或图片需要 OCR，不要猜测其内容。没有可用 OCR 产物时，记录 OCR 需求并暂停，或请求所需 OCR 产物。

OCR 或解析服务所需 token 和凭据必须来自环境变量或主机配置。不要把 token 写入 source indexes、fact packets、validation reports、日志或仓库文件。

## 冲突规则

如果允许来源之间相互矛盾，应保留冲突。除非 active profile 声明了明确优先级规则且被选证据满足该规则，否则不得静默选择其中一个值。
