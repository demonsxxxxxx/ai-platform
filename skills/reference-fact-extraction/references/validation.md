# 校验策略

校验用于保护下游用户，避免消费外观完整但没有证据支撑的事实。

## 消费门禁

事实包只有在满足以下条件时才可被消费：

- schema 校验通过；
- 所有被消费的 shards 都有 passing validation status；
- 已记录由 `subagent` 执行的独立核验；
- 必需 profile 字段存在，或已明确列入 missing evidence；
- 未解决冲突要么按 profile 策略属于非阻断项，要么已暴露给人工审查；
- 没有事实引用禁止来源；
- 没有事实仅依赖未审查的 `source-index` preview。

## 没有证据就没有事实

如果某个值未找到、不确定、被反驳或超出抽取者来源访问范围，应记录为：

- 需要的证据缺失时，写入 `missing_evidence`；
- 需要人工决策时，写入 `manual_review_items`；
- 多个来源不一致时，写入 `conflicts`；
- 拟采纳事实未通过核验时，写入 `rejected_facts`。

不要用看似合理的值填补缺失字段。

## 独立核验

核验必须检查来源支撑，而不只是检查 JSON 形状。核验者应尝试反驳该分片：

- 来源是否允许？
- 引用的文本或表格单元格是否真的支持该值？
- 单位、条件、限定语、日期和范围是否被保留？
- 是否忽略了矛盾？
- 推导事实是否可复现？
- profile 特定规则是否满足？

## Subagent 门禁

`agent_provenance.execution_mode` 必须是 `subagent`。`same_thread_separated_pass` 只能用于调试或诊断，不能通过 shard validation，不能装配为可消费事实包，也不能交给下游渲染或写作 skill。

## 警告

只有当警告是显式、可追溯且不改变必需事实时，才可接受。影响必需事实或证据链的警告应阻断最终消费。

## 下游渲染

下游写作或渲染只能消费已验证事实包。叙述文本可以改写事实表达，但不得引入新事实。
