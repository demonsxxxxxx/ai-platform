# 写作模式

在事实包完成后使用这些模式。措辞必须根据项目证据进行调整。

## 目录

- `语义占位符`：正文槽位、写作经验引用和使用边界。
- `真实结果稿经验沉淀`：去事实化段落骨架和常见场景。
- `章节骨架`：3.2.S.7.3 正文组织顺序。
- `长期稳定性`、`加速稳定性`：正文结论写法和边界。
- `影响因素研究引言`、`单项影响因素结果模式`、`各影响因素试验注意事项`：影响因素段落写法。
- `正文中的表格和图`、`趋势图生成`：表图引用和趋势图文字规则。
- `禁止的捷径`：不得复用的模板化或无证据表达。

## 语义占位符

模板正文使用整段语义占位符。状态机进入 `BODY_SECTIONS_REQUIRED` 后，先读取 `body-sections-request.json` 和本文，再在事实包 `body_sections` 中写入最终段落文本；骨架生成阶段保留这些占位符，pipeline 在表格渲染前统一替换。

槽位和参考写作经验：

| 槽位 | 参考写作经验 | 起草重点 |
| --- | --- | --- |
| `{{LONG_TERM_INTRO}}` | 长期稳定性 | 写清产品 / 样品类型、批次角色、长期条件、结果表范围、已完成月份和是否继续。 |
| `{{LONG_TERM_TREND_INTRO}}` | 长期稳定性；趋势图生成 | 只有对应趋势图已生成且编号确认后，才引用图范围和原始图谱资料名。 |
| `{{LONG_TERM_TREND_SUMMARY}}` | 长期稳定性；禁止的捷径 | 按实际检项趋势写结论；不同批次或检项表现不一致时分别说明。 |
| `{{ACCELERATED_INTRO}}` | 加速稳定性 | 写清加速条件、批次、表范围、完成月份和是否已完成研究。 |
| `{{ACCELERATED_TREND_INTRO}}` | 加速稳定性；趋势图生成 | 确认加速趋势图产物和编号后，再写图范围和图谱资料名。 |
| `{{ACCELERATED_TREND_SUMMARY}}` | 加速稳定性；禁止的捷径 | 短期挑战条件下如有变化，必须写明检项、方向和是否仍在标准范围内。 |
| `{{STRESS_STUDY_INTRO}}` | 影响因素研究引言 | 写清模拟场景、样品形态、试验项列表，并说明影响因素研究关注变化趋势。 |
| `{{STRESS_LIGHT_REFERENCE}}` | 正文中的表格和来源图谱；光照注意事项 | 用于光照表格或来源图谱引用；影响因素研究不生成趋势图，不能写成“见趋势图”。 |
| `{{STRESS_LIGHT_SUMMARY}}` | 单项影响因素结果模式；光照注意事项 | 按光照数据判断是否敏感；不要在证据不足时写“对光敏感”。 |
| `{{STRESS_AGITATION_SUMMARY}}` | 单项影响因素结果模式；振荡注意事项 | 写清转速、温度、避光和时间；按数据判断耐受性及运输 / 操作建议。 |
| `{{STRESS_FREEZE_THAW_SUMMARY}}` | 单项影响因素结果模式；反复冻融注意事项 | 写清循环次数、冻结 / 融化条件，并给出减少反复冻融的控制建议。 |
| `{{STRESS_HIGH_TEMPERATURE_SUMMARY}}` | 单项影响因素结果模式；高温注意事项 | 写清温度、湿度、避光和时间；只有数据支持时才写轻微或明显敏感。 |
| `{{STRESS_PH_SUMMARY}}` | 单项影响因素结果模式；pH 注意事项 | 低 pH 和高 pH 原则上分别描述；如合并总结，必须保留两者差异。 |
| `{{STRESS_OXIDATION_SUMMARY}}` | 单项影响因素结果模式；氧化注意事项 | 写清氧化剂和浓度，关注电荷、活性、氧化位点、DAR、游离载荷和蛋白含量。 |
| `{{FINAL_STABILITY_CONCLUSION}}` | 章节骨架；各影响因素试验注意事项；禁止的捷径 | 汇总长期、加速和影响因素结论；不得引入事实包外的新判断。 |

每个槽位的 `text` 应是可直接进入 DOCX 的完整段落。不要把模板备选词、旧项目事实或缺证据推断写进 `body_sections`。

正文阶段必须在事实包中为每个正文槽位记录 `writing_pattern_refs`，例如 `["真实结果稿经验沉淀.LONG_TERM_INTRO.ongoing"]`。该字段只记录采用的写作模式，不替代 `source_refs`，也不能作为项目事实依据。

如果事实场景难以只靠模式判断，读取 `references/writing-fact-cases.md`。案例库提供“事实快照 → 应写段落 → 不应写”的去事实化示例；采用案例时可在 `writing_pattern_refs` 追加 `事实案例.CASE-...`。

## 真实结果稿经验沉淀

本节由真实结果稿语料抽象而来，包括 IP177、IP248A、IP315、IP319、IP350 项目下的 `已完成的申报资料...docx`。这些文件只用于沉淀段落组织、连接词、结论边界和常见场景；不得把其中的项目名、批号、条件、数值、图号、表号或判断迁移到当前项目。默认不要在运行时读取已完成申报资料；只有用户明确批准且该文件未列为 forbidden source 时，才可作为写作模式语料。示例均已去事实化，方括号内容必须由当前 `fact-packet.json` 提供。

起草 `body_sections` 时按槽位写完整段落，不做零散词替换。先判断场景，再选示例骨架：

- `LONG_TERM_INTRO.completed`：长期研究已完成时，写“对[产品表达]（批号：[批号列表]）进行了[长期条件][、避光]条件下的长期稳定性[研究/考察]，[研究方案见表X，]考察结果见[表范围]。结果[表明/显示]，[已完成月份]个月的各项检测结果均符合[质量标准/可接受标准]。”不要再写“将持续考察”。
- `LONG_TERM_INTRO.ongoing`：长期研究未完成时，写“对[产品表达]（[非临床批/临床批角色和批号]）进行了[长期条件][、避光]条件下的长期稳定性考察，考察结果见[表范围]。结果表明，[批次A]已完成[月数]个月、[批次B]已完成[月数]个月的各项检测结果均符合[标准]，该长期稳定性研究将持续考察至[计划月数]个月。”如果各批月份一致，可合并为“[批次数量]批次[月数]个月”。
- `LONG_TERM_TREND_INTRO.with_figures`：有趋势图时，写“在[长期条件][、避光]长期稳定性研究条件下，[批次和已完成月份]的各检项稳定性趋势分析见[图范围]。原始的稳定性图谱见本品申报资料‘[资料名]’。”只有 `trend-chart-manifest.json` 确认图号后才引用图范围；没有图谱资料名时删除第二句。
- `LONG_TERM_TREND_SUMMARY.stable`：趋势稳定时，写“结果表明：[批次数量][产品表达]的[SEC 检项]（[图号]）、[CE-SDS 检项]（[图号]）、[电荷变异体检项]（[图号范围]）、[活性/蛋白含量/pH]（[图号]）结果均呈现较为稳定的趋势，[并均符合[样品形态]质量标准的相关要求]。”括号内图号只在真实存在时保留；不要列入未成图检项。
- `LONG_TERM_TREND_SUMMARY.limited_data`：只有起始点或单个有效后续时间点时，不写“趋势稳定”，改写为“结果表明：[产品表达]长期稳定性研究已完成时间点的检测结果未见明显不利变化；后续趋势判断将在更多时间点完成后继续更新。”该句仍必须由事实包支持。
- `ACCELERATED_INTRO.stable`：加速数据稳定时，写“对[产品表达]（批号：[批次列表]）进行了[加速条件][、避光]条件下的加速稳定性考察，考察结果见[表范围]。结果表明，在[加速条件]条件下，[批次和已完成月份]各项检测结果无明显趋势变化，均保持稳定[，符合[标准]]。”计划未完成时补“该加速稳定性研究将持续考察至[计划月数]个月”；已完成时不要补继续考察。
- `ACCELERATED_INTRO.changed`：加速下存在变化时，写“结果表明，在[加速条件]条件下，[批次A][月数]个月[检项1]略有[上升/下降]、[检项2]略有[波动]，其余各检项结果均保持稳定，[变化检项]仍符合[标准]；[批次B][月数]个月[差异表现]。”不同批次表现不同必须分批写。
- `ACCELERATED_TREND_INTRO.with_figures`：与长期趋势引言同构，但图号应从加速图组读取，例如“在[加速条件][、避光]加速稳定性研究条件下，[批次和月份]各检项结果的趋势分析见[图范围]。原始的稳定性图谱见本品申报资料‘[资料名]’。”不要沿用长期图号。
- `ACCELERATED_TREND_SUMMARY.changed`：短期挑战条件下如出现变化，写“结果表明：[稳定检项列表]趋势较为稳定；[变化检项]趋势显示，[具体组分/指标]含量[上升/下降/波动]，[如适用：但均符合[标准]的相关要求]。”不要用“均稳定”覆盖变化。
- `ACCELERATED_TREND_SUMMARY.limited_data`：加速研究仅有起始点或有效时间点不足时，写“结果表明：[产品表达]加速稳定性研究目前仅完成[时间点描述]，趋势判断需结合后续时间点继续评估。”不要提前下“稳定”结论。
- `STRESS_STUDY_INTRO.actual_tests`：写“模拟[样品形态]在贮存、运输、生产过程中可能遇到的剧烈条件来设计[样品形态]的影响因素试验，包括[实际开展项列表]。影响因素试验不设定可接受标准，主要关注各检项检测结果的变化趋势。”只列 `stress_study.included_tests` 中存在的试验。
- `STRESS_LIGHT_REFERENCE`：只写“光照试验结果[见/如][表号]所示。[原始的稳定性图谱见本品申报资料‘[资料名]’。]”影响因素研究不生成趋势图；没有图谱资料名时不要补图谱来源。
- `STRESS_LIGHT_SUMMARY.sensitive`：明显变化时，写“结果表明，[样品]在[温湿度、照度、时间]条件下放置后，[SEC 单体下降/聚体上升]；[CE-SDS 主峰下降/片段或低分子量组分上升]；[电荷变异体酸性组分上升、主峰下降]；[活性/DAR/游离载荷等变化]；其他检项无明显变化。因此，[样品]对光照条件[较为/非常]敏感。为保证样品稳定性，建议在生产、储存和转运等过程中避免暴露在较强光线下。”
- `STRESS_LIGHT_SUMMARY.mild_or_unclear`：变化较小或来源未作敏感判断时，写“光照试验结果表明：[样品]在[条件]下[主要轻微变化]，其他检项无明显变化。为保证样品稳定性，建议[避光/避免强光照射]。”不要写“对光敏感”。
- `STRESS_AGITATION_SUMMARY.no_change`：无明显变化时，写“振荡试验结果见[表号]。[样品]在[温度、转速、避光、时间]条件下振荡后，各项检测结果[无明显变化/仅[检项]轻微变化]，表明[样品]对振荡具有一定耐受性。为保证产品稳定性，生产、储存、转运过程中仍应避免剧烈振荡。[图谱来源]”
- `STRESS_AGITATION_SUMMARY.changed`：变化明显时，写“振荡试验结果表明：[样品]在[条件]下振荡[时间]后，[SEC/电荷/澄清度/活性等]发生明显变化，[其余检项]无明显变化。因此，为保证样品稳定性，建议生产、储存和转运过程中避免剧烈振动。”
- `STRESS_FREEZE_THAW_SUMMARY.no_change`：写“[样品]经[循环次数]次冻融（[冻结条件]，[融化条件]）后，[各项检测结果/所有检项结果]均无明显变化，表明[样品]对反复冻融具有一定耐受性。为保证产品稳定性，仍应尽量减少多次反复冻融。[图谱来源]”
- `STRESS_PH_SUMMARY.split_low_high`：低 pH 和高 pH 分别写变化，再合并建议，例如“pH [低值]条件下，[SEC/CE-SDS/电荷/活性/蛋白含量变化]；pH [高值]条件下，[对应变化]；其余检测结果无明显变化。[若来源支持：蛋白浓度偏低与酸/碱加入导致稀释有关。]因此，建议[样品]在生产、储存、运输过程中避免极端 pH 条件或酸碱试剂接触。[图谱来源]”
- `STRESS_OXIDATION_SUMMARY.adc_or_antibody`：写“氧化试验结果表明：[样品]在[氧化剂、浓度、温湿度、时间]条件下放置后，[电荷变异体/CE-SDS/活性/氧化位点/DAR/游离小分子药物/蛋白含量]发生[具体变化]；其余检项[无明显变化/变化较小]。因此，[样品]对氧化剂[较为/极度]敏感，[或：应避免接触潜在氧化剂]，生产、储存和运输过程中应避免接触潜在氧化剂并保持容器密闭。”
- `STRESS_HIGH_TEMPERATURE_SUMMARY.changed`：写“高温试验结果显示：[样品]在[温度、湿度、避光、时间]条件下放置后，[SEC/CE-SDS/电荷/活性/游离载荷/蛋白含量]发生[具体变化]，其余检项无明显变化。因此，[样品]对高温环境[略微/较为]敏感，建议生产、储存、转运过程中尽量避免高温环境。”如果来源只是“对照试验”或条件接近室温，按来源名称改写，不机械称为高温敏感。
- `FINAL_STABILITY_CONCLUSION.actual_tests_only`：写“综上所述，[产品表达]在[长期/加速条件]下[稳定性概括]；在生产、储存和运输过程中应[按实际影响因素结果列出控制建议]，并[保持容器密闭/避光/避免强光/避免极端 pH/避免潜在氧化剂等]，从而保证[样品形态]的稳定性。”最终结论只汇总已经开展且前文写过的影响因素；未开展振荡时不得写“减少剧烈振荡”，未开展高温时不得写“避免高温环境”。

结果稿中的病句、笔误和未收束表达都视为反例，不要复用。例如“结果显示，结果显示”、单位缺字、“各项检测结果各检项结果”、只有“结果显示。”却没有结论、未开展试验仍进入最终结论、以及光照/氧化等来源未支持却强行写“敏感”。

## 章节骨架

典型的 3.2.S.7.3 结构：

1. 长期稳定性结果。
2. 加速稳定性结果。
3. 影响因素 / 强制降解研究。

影响因素研究小节由 pipeline 根据 `stress_study.included_tests` 和 `stress_study.table_render_inputs[].role` 动态生成。可生成的小节包括：

- 研究引言。
- 光照。
- 振荡。
- 反复冻融。
- 高温。
- 低 pH。
- 高 pH。
- 氧化。
- 影响因素研究最终结论。

目标样品类型未开展的影响因素研究小节不会生成。起草 `body_sections` 时应只给实际开展试验编写结果段落，最终结论也不得保留未开展试验的控制建议。

## 长期稳定性

长期稳定性引言应回答：

- 研究了什么产品 / 样品类型。
- 研究了哪些批次及其角色。
- 使用了什么条件。
- 哪些表格包含结果。
- 各批次已完成哪些月份。
- 研究是否将继续。

模式：

> 对[产品表达]（[批次角色和批号列表]）进行了[长期条件][、避光]条件下的长期稳定性考察，考察结果见[表范围]。结果表明，[各批已完成月份]各项检测结果均符合[质量标准/可接受标准]，该长期稳定性研究将持续考察至[计划月份]个月。

如果已完成月份不同：

> 结果表明，非临床批[批号]已完成[月数]个月、临床批[批号]已完成[月数]个月，各项检测结果均符合可接受标准。

趋势段落：

> 在[长期条件]长期稳定性研究条件下，[批号和月份]的各检项稳定性趋势分析见[图范围]。原始的稳定性图谱见本品申报资料“[图谱资料名]”。

趋势结论：

> 结果表明：[批次数量][产品表达]的[检项列表]结果整体呈现较为稳定的趋势，[如有变化则写具体变化]，均符合[样品形态]质量标准的相关要求。

## 加速稳定性

加速章节应重点说明短期挑战条件下的趋势变化。

模式：

> 对[产品表达]（批号：[批次列表]）进行了[加速条件]条件下的加速稳定性考察，考察结果见[表范围]。结果表明，在[加速条件]条件下，[批次和月份]各项检测结果[无明显趋势变化/除某检项外均保持稳定]。

规则：

- 如果所有检测项目均稳定，应写无明显趋势变化，并说明仍符合标准。
- 如果某检测项目发生变化，应写明检测项目和变化方向。
- 如果变化仍在标准范围内，应说明这一点。
- 如果计划的加速研究已完成，不要写仍将继续。

## 影响因素研究引言

模式：

> 模拟[样品形态]在贮存、运输、生产过程中可能遇到的剧烈条件来设计[样品形态]的影响因素试验，包括[试验项列表]。影响因素试验不设定可接受标准，主要关注各检项检测结果的变化趋势。

## 单项影响因素结果模式

使用以下逻辑链：

> 条件和时间 → 主要变化 → 其他检项 → 敏感性/耐受性判断 → 控制建议 → 来源图谱

模式：

> [试验项]结果见[表号]。结果显示，[产品表达]在[条件]下放置[时间]后，[主要变化]；[其他检项]无明显变化。因此，[敏感性/耐受性判断]。[控制建议]。原始的稳定性图谱见本品申报资料“[资料名]”。

## 各影响因素试验注意事项

光照：

- 常见变化：SEC 单体下降、聚体 / 片段上升、电荷异构体变化、活性变化。
- 建议通常与避免强光照射有关。
- 如果变化很小且报告不支持，不要称产品对光敏感。

振荡：

- 写明转速、温度、避光要求和持续时间。
- 如果出现明显 SEC / 电荷 / 活性变化，建议避免剧烈振摇。
- 如果无明显变化，可使用“具有一定耐受性”，但仍应建议尽量减少振荡。

反复冻融：

- 写明循环次数以及冻结 / 融化条件。
- 如果无明显变化，写“对反复冻融具有一定耐受性”。
- 仍应建议减少反复冻融。

pH：

- 除非最终总结有意合并，否则低 pH 和高 pH 必须分别描述。
- 写明目标 pH、酸 / 碱、是否调回 pH 以及持续时间。
- 如果酸 / 碱加入导致稀释并影响蛋白浓度，且来源支持，应说明这一点。

氧化：

- 写明氧化剂和浓度。
- 关注电荷异构体、活性、氧化位点、DAR、游离载荷杂质和蛋白含量。
- 如果数据支持敏感性，建议避免接触氧化剂并保持容器密闭。

高温：

- 写明温度、湿度、避光要求和持续时间。
- 只有在数据支持时，才写产品轻微敏感或明显敏感。

## 正文中的表格和图

- 一个批次通常对应一个长期表和一个加速表。
- 每个影响因素试验通常每个批次对应一个表。
- 单批次项目应使用单表引用，不要写成范围引用，也不要暗示存在第二批。
- 题注应包含样品类型、批次、条件和试验类型。
- 图引用必须与生成的长期 / 加速趋势图编号一致。
- 正文里的表号和图号必须来自本项目最终渲染编号，不要把来源文件里的局部表号直接搬进正文。长期 / 加速段落只引用对应研究组的渲染表号和趋势图号，影响因素段落只引用对应 role 的渲染表号。
- 影响因素研究不生成趋势图；如需提及图谱 / 色谱图，只能写为来源资料引用，不写入 `trend_charts`。
- 如果未生成某个长期或加速趋势图，应删除引用该图的文字。
- 长期 / 加速趋势图应覆盖事实包中按批次和时间点整理、且具备 `0 月 + 后续时间点` 数值结果的检项；不要只画一个代表检项。

## 趋势图生成

需要长期或加速趋势图时，在状态机进入 `TREND_CHARTS_REQUIRED` 后，在同一个事实包中提供趋势图输入契约，并通过 step 让内部 runtime 生成或纳入图表产物。不要直接调用内部趋势图脚本。影响因素研究不需要趋势图，也不应进入 `trend_charts.charts[]`。

写作和题注规则：

- 正文中写“趋势分析见图...”前，先确认对应 PNG 已生成且 manifest 无未解释 warning。
- 长期图和加速图应分组编号，并与正文图范围一致。
- 正文图范围、趋势结论列举的检项和 `trend-chart-manifest.json` 必须一致。
- 趋势图引用优先使用 manifest 驱动令牌，例如 `{{LONG_TERM_TREND_FIGURE_RANGE}}`、`{{ACCELERATED_TREND_FIGURE_RANGE}}` 或当前段落上下文中的 `{{TREND_FIGURE_RANGE}}`，避免在事实包抽取阶段手写易失效的图号范围。
- 未来未到时间点、`---`、`N/A`、空白和非数值描述不得作为趋势点。
- 若某图只有一个有效时间点，仍可生成点图，但验证报告应说明趋势判断受限；若没有有效数值，不要保留该图引用。
- 双指标效价图可用双面板图呈现，不要把 hACVR2A 和 hACVR2B 混成同一条趋势线。

## 禁止的捷径

- 数据显示明显变化时，不要写“均稳定”。
- 不要保留“略微上升/略微下降/波动”等模板选项；必须根据数据选择其一。
- 不要把待完成研究描述为已完成。
- 除非来源文件说明适用可接受标准，否则不要把影响因素研究写成符合标准的研究。
- 不要从其他项目的已完成申报资料中导入事实。
